# Dinner Page Mobile UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the dinner cook page's quick-jump nav non-sticky and apply a tight set of mobile-friendly CSS polish, without changing behavior, data, or JS.

**Architecture:** Pure CSS edits to two stylesheets (`DinnerChecklist.css`, `global.css`). No component logic or markup changes. Verified by a green build plus assertions on the built CSS, then a manual responsive check.

**Tech Stack:** Astro 4 (static), plain CSS. No tests framework involved (CSS-only).

## Global Constraints

- **CSS only** — no JavaScript, no `.tsx`/`.astro` markup or logic changes, no content/schema changes.
- **No regressions** to the recipe or shopping-list pages, which share `global.css`.
- **No font shrink** below current sizes; legibility preserved.
- Do NOT touch `vitest.config.ts`, `.gitignore`, or unrelated files. The untracked `refrences/` folder is pre-existing — leave it.

### Task 1: Non-sticky jump nav + mobile polish

**Files:**
- Modify: `src/components/DinnerChecklist.css`
- Modify: `src/styles/global.css`

**Interfaces:**
- Consumes: existing class names from `DinnerChecklist.tsx` (`.dinner`, `.jump`, `.dish`, `.checklist`, `.checklist label`, `.checklist input`) and the `header.site .container` markup in `BaseLayout.astro`. No markup changes — only styling those existing elements.
- Produces: nothing consumed by other code; this is the final task.

- [ ] **Step 1: Remove sticky positioning from the jump nav**

In `src/components/DinnerChecklist.css`, replace the `.dinner .jump` rule (currently line 2):

```css
.dinner .jump { display: flex; flex-wrap: wrap; gap: 0.5rem; align-items: center; position: sticky; top: 0; background: var(--bg); padding: 0.5rem 0; border-bottom: 1px solid var(--border); }
```

with (drop `position: sticky; top: 0;`):

```css
.dinner .jump { display: flex; flex-wrap: wrap; gap: 0.5rem; align-items: center; background: var(--bg); padding: 0.5rem 0; border-bottom: 1px solid var(--border); }
```

- [ ] **Step 2: Bigger tap targets + text wrapping in the checklist**

In `src/components/DinnerChecklist.css`, replace the `.checklist li` rule (currently line 12):

```css
.checklist li { padding: 0.4rem 0; border-bottom: 1px solid var(--border); font-size: 1.05rem; }
```

with (more vertical padding for easier tapping):

```css
.checklist li { padding: 0.55rem 0; border-bottom: 1px solid var(--border); font-size: 1.05rem; }
```

Then replace the `.checklist input` rule (currently line 14):

```css
.checklist input { width: 1.15rem; height: 1.15rem; flex: none; }
```

with (larger touch target):

```css
.checklist input { width: 1.4rem; height: 1.4rem; flex: none; }
```

Then add this new rule immediately after the `.checklist label` rule (after line 13) so the text column can shrink and long words wrap instead of overflowing:

```css
.checklist label > span { min-width: 0; overflow-wrap: anywhere; }
```

- [ ] **Step 3: Anchor breathing room + small-screen tuning**

Append the following to the END of `src/components/DinnerChecklist.css`:

```css
.dish { scroll-margin-top: 0.75rem; }

@media (max-width: 520px) {
  .dinner { gap: 1rem; }
  .dish { padding: 0.85rem 0.9rem; }
}
```

- [ ] **Step 4: Header nav wraps + global overflow safety**

In `src/styles/global.css`, replace the `header.site .container` rule (currently line 14):

```css
header.site .container { display: flex; gap: 1rem; align-items: baseline; }
```

with (allow wrapping on narrow screens):

```css
header.site .container { display: flex; flex-wrap: wrap; gap: 0.5rem 1rem; align-items: baseline; }
```

Then replace the `body` rule (currently line 10):

```css
body { margin: 0; background: var(--bg); color: var(--fg); line-height: 1.5; }
```

with (prevent long words/strings from forcing horizontal scroll, and stop iOS text auto-zoom):

```css
body { margin: 0; background: var(--bg); color: var(--fg); line-height: 1.5; overflow-wrap: break-word; -webkit-text-size-adjust: 100%; }
```

- [ ] **Step 5: Build and assert the CSS changes landed**

Run: `npm run build`
Expected: build succeeds (28 pages, no errors).

Then verify the intended CSS is in the built output and the sticky rule is gone. Run (Git Bash):
```bash
grep -rlE "position: ?sticky" dist/ || echo "NO sticky in dist (good)"
grep -roE "scroll-margin-top" dist/ | head -1
grep -roE "width: ?1.4rem" dist/ | head -1
```
Expected: "NO sticky in dist (good)" prints (neither `position:sticky` nor `position: sticky` remains in the built CSS — the `-E` regex matches both minified and unminified forms); `scroll-margin-top` and the `1.4rem` checkbox width are present.

- [ ] **Step 6: Commit**

```bash
git add src/components/DinnerChecklist.css src/styles/global.css
git commit -m "style: make dinner jump-nav non-sticky and improve mobile UX"
```
